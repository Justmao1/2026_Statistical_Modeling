# OpenClaw 数据采集提示词（统计建模大赛专用）

> **项目：** 数字经济对区域碳排放强度的影响研究
> **数据范围：** 中国30个省级行政区（除港澳台及西藏），2011-2023年
> **输出要求：** 所有数据统一保存为 CSV 格式，编码 UTF-8，列名含中英文

---

## 📋 使用说明

以下每个提示词对应一类数据采集任务，按顺序逐个发送给 OpenClaw 执行即可。每个提示词都明确了：
- 目标网站 / 数据源
- 需要采集的具体字段
- 输出文件名和格式要求

> [!WARNING]
> **重要提醒：** 部分数据源（如CEADs）需要注册账号后才能下载，请提前完成注册。国家统计局数据可直接公开获取。

## 提示词 1：碳排放数据（因变量）

```
请帮我从 CEADs 中国碳核算数据库（https://www.ceads.net.cn/）采集以下数据：

数据范围：中国30个省级行政区（不含港澳台和西藏），2011年至2023年

需要采集的字段：
1. 省份名称
2. 年份
3. CO2排放总量（万吨）
4. 分部门碳排放数据（如有）：工业、交通运输、建筑、电力等

操作步骤：
- 进入官网，找到"数据下载"或"省级清单"栏目
- 下载各年份的省级碳排放数据
- 如果需要登录，请提醒我手动登录后再继续

输出要求：
- 保存为 CSV 文件，文件名：01_碳排放数据_CEADs.csv
- 列名格式：省份, 年份, CO2排放总量_万吨
- 确保数据按"省份+年份"排列，形成面板数据结构
```

## 提示词 2：GDP与经济基础数据（因变量计算 + 控制变量）

```
请帮我从国家统计局官网（https://data.stats.gov.cn/）采集以下省级年度数据：

数据范围：中国30个省级行政区（不含港澳台和西藏），2011年至2023年

需要采集的字段（请逐项查找）：
1. 地区生产总值 GDP（亿元）
2. 人均地区生产总值（元）
3. 年末常住人口（万人）
4. 城镇常住人口占比（%）——即城镇化率
5. 第一产业增加值（亿元）
6. 第二产业增加值（亿元）
7. 第三产业增加值（亿元）

操作路径建议：
- 进入"国家数据" → "年度数据" → "分省年度数据"
- 依次在"国民经济核算"、"人口"等分类下查找对应指标
- 选择所有省份、2011-2023年时间范围，导出数据

输出要求：
- 保存为 CSV 文件，文件名：02_GDP与经济数据_国家统计局.csv
- 列名格式：省份, 年份, GDP_亿元, 人均GDP_元, 常住人口_万人, 城镇化率_百分比, 第一产业增加值_亿元, 第二产业增加值_亿元, 第三产业增加值_亿元
- 数据按"省份+年份"排列
```

## 提示词 4：数字产业与软件数据（自变量构建）

```
请帮我从工信部官网（https://www.miit.gov.cn/）及国家统计局采集以下省级数据：

数据范围：中国30个省级行政区，2011年至2023年

需要采集的字段：
1. 软件业务收入（亿元）——按省份
2. 信息技术服务收入（亿元）——按省份
3. 电子信息制造业主营业务收入（亿元）——如有省级数据

操作建议：
- 工信部每年发布《软件和信息技术服务业统计公报》，其中包含分省数据

- 如果工信部上找不到历年分省数据，可以尝试在国家统计局搜索"软件业务收入"相关指标

- 保存为 CSV 文件，文件名：04_数字产业数据.csv
- 列名格式：省份, 年份, 软件业务收入_亿元, 信息技术服务收入_亿元
- 如果某些年份数据缺失，请在对应单元格标记为 NA，并在文件末尾备注说明
```

## 提示词 6：能源消费数据（控制变量 + 中介变量）

```
请帮我采集中国各省级行政区的能源消费数据。

数据来源优先级：
1. 《中国能源统计年鉴》（可在国家统计局或知网下载）
2. 国家统计局官网（https://data.stats.gov.cn/）
3. 各省统计年鉴

数据范围：30个省级行政区（不含港澳台和西藏），2011-2023年

需要采集的字段：
1. 能源消费总量（万吨标准煤）
2. 煤炭消费量（万吨）或煤炭消费占一次能源比例（%）
3. 单位GDP能耗（吨标准煤/万元）——如有直接指标

操作建议：
- 在国家统计局"分省年度数据"中搜索"能源消费"
- 查找"分地区能源消费总量"和"分地区煤炭消费量"
- 如果找不到煤炭消费占比，我们可以用"煤炭消费量/能源消费总量"自行计算

输出要求：
- 保存为 CSV 文件，文件名：06_能源消费数据.csv
- 列名格式：省份, 年份, 能源消费总量_万吨标准煤, 煤炭消费量_万吨, 单位GDP能耗
- 缺失数据标记为 NA
```

## 提示词 13：数据汇总与校验

```
我已经通过前面的任务采集了12个数据文件。现在请帮我做以下数据校验工作：

1. 检查所有CSV文件是否都覆盖了30个省份 × 13年（2011-2023），理论上每个文件应有390行数据（面板结构）
2. 列出每个文件中的缺失值情况（哪些省份、哪些年份有 NA）
3. 检查省份名称是否统一（有些文件可能用"内蒙古"，另一些用"内蒙古自治区"）
4. 生成一份数据完整性报告

请将校验结果整理为一份 Markdown 报告，保存为：
文件名：数据完整性报告.md

报告结构：
- 各文件概况（行数、列数、缺失比例）
- 省份名称一致性检查
- 关键缺失数据的处理建议
```

## 📌 采集顺序建议

| 优先级 | 提示词编号 | 数据类型 | 预计难度 |
|--------|-----------|---------|---------|
| ⭐️⭐️⭐️ | 2 | GDP与经济数据 | 低（国家统计局直接获取） |
| ⭐️⭐️⭐️ | 3 | 数字基础设施 | 低 |
| ⭐️⭐️⭐️ | 1 | 碳排放数据 | 中（需注册CEADs账号） |
| ⭐️⭐️ | 5 | 数字普惠金融指数 | 低（有现成数据集） |
| ⭐️⭐️ | 6 | 能源消费数据 | 中 |
| ⭐️⭐️ | 7 | FDI数据 | 低 |
| ⭐️⭐️ | 8 | 环境规制数据 | 中 |
| ⭐️⭐️ | 9 | 专利与科技数据 | 低 |
| ⭐️ | 4 | 数字产业数据 | 高（分散在工信部各年公报） |
| ⭐️ | 10 | 绿色专利数据 | 高（可能需要手工检索） |
| ⭐️ | 11 | 历史工具变量 | 中（历史数据较难找） |
| ⭐️ | 12 | 碳交易试点信息 | 低（信息明确） |
| 最后 | 13 | 数据汇总校验 | — |

> [!TIP]
> **建议先从优先级最高的（提示词2、3）开始**，这些数据在国家统计局可以直接批量导出，成功率最高。碳排放数据（提示词1）需要先去 ceads.net.cn 注册账号。

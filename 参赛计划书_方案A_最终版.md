## 一、选题背景与研究意义

### 1.1 现实背景

中国已明确提出"2030年前碳达峰、2060年前碳中和"的双碳战略目标。与此同时，数字经济已成为推动高质量发展的核心引擎——2025年中国数字经济规模超55万亿元，占GDP比重超40%。

**核心问题：** 数字经济能否有效助力碳减排？其作用机制是什么？在不同发展阶段的地区，减碳效果有何差异？我们能否通过机器学习实现更精准的碳排放预测与关键因素识别？

### 1.2 研究意义

| 维度 | 具体意义 |
|------|---------|
| **理论意义** | 将数字经济纳入STIRPAT碳排放分析框架，拓展环境库兹涅茨曲线(EKC)理论 |
| **方法创新** | 构建"统计因果推断 + ML预测 + SHAP解释"的三位一体分析框架 |
| **政策价值** | 为双碳目标下区域差异化数字减碳政策提供量化决策支撑 |

### 1.3 契合大赛主题

- **"服务国家战略"**：直接对接"双碳"目标 + 数字中国战略
- **"创新统计赋能"**：传统统计因果推断 + 现代AI预测 + SHAP可解释性，体现"创新"

### 1.4 预期创新点（三点，论文引言中明确写出）

1. **方法融合创新**：构建"空间计量（因果推断）+ 机器学习（精准预测）+ SHAP（可解释AI）"的三位一体分析框架，突破传统单一方法的局限
2. **数据多维创新**：基于熵权法自行构建4维12指标的数字经济综合指数，而非依赖单一代理变量
3. **政策参考创新**：结合SHAP特征重要性排序与异质性分析（碳交易试点 vs 非试点），为差异化政策设计提供AI辅助的精准统计依据

### 3.2 Part A：统计建模（因果推断）

#### 模型一：基准回归（双向固定效应）

$$CO2_{it} = \alpha_0 + \alpha_1 DE_{it} + \alpha_2 DE_{it}^2 + \gamma X_{it} + \mu_i + \delta_t + \varepsilon_{it}$$

- $CO2_{it}$：省份i在年份t的碳排放强度
- $DE_{it}$：数字经济综合指数
- $DE_{it}^2$：二次项（检验倒U型）
- $\mu_i$：省份固定效应，$\delta_t$：时间固定效应

#### 模型二：空间杜宾模型（SDM）

$$CO2_{it} = \rho W \cdot CO2_{it} + \beta_1 DE_{it} + \theta_1 W \cdot DE_{it} + \gamma X_{it} + \mu_i + \delta_t + \varepsilon_{it}$$

- 检验Moran's I空间自相关 → LM检验选择模型 → SDM估计
- 分解直接效应与间接效应（空间溢出）

#### 模型三：Hansen门槛效应

$$CO2_{it} = \beta_1 DE_{it} \cdot I(q_{it} \leq \gamma_1) + \beta_2 DE_{it} \cdot I(q_{it} > \gamma_1) + \delta X_{it} + \mu_i + \varepsilon_{it}$$

- 门槛变量：人均GDP / 城镇化率
- Bootstrap检验门槛存在性（300次自抽样）

#### 模型四：中介效应（三步法）

```
路径一: DE → 产业结构高级化(ISU) → 碳排放强度↓
路径二: DE → 绿色技术创新(GTI) → 碳排放强度↓
路径三: DE → 能源利用效率(EE)  → 碳排放强度↓

每条路径均进行Sobel检验 + Bootstrap中介效应检验
```

### 3.3 Part B：机器学习预测与SHAP分析（核心新增）

#### B1. 特征工程

```python
特征矩阵 X 包含：
├── 数字经济子指标（12个二级指标，非综合指数）
│   ├── 互联网普及率、移动电话普及率、光缆长度
│   ├── 电信业务总量、IT服务收入、软件业务收入
│   ├── 数字普惠金融指数（覆盖广度、使用深度、数字化程度）
│   └── 数字专利数、R&D人员密度、技术市场成交额
├── 控制变量（6个）
│   ├── 人均GDP、产业结构、城镇化率
│   └── FDI、环境规制、能源消费结构
└── 交互特征（可选）
    └── DE × 人均GDP、DE × 产业结构

目标变量 y：碳排放强度（CO2/GDP）
```

#### B2. 模型训练与调参

| 模型 | 关键超参数 | 调参方法 |
|------|----------|---------|
| 随机森林(RF) | n_estimators, max_depth, min_samples_split | 5折交叉验证 + GridSearch |
| XGBoost | learning_rate, max_depth, n_estimators, reg_alpha | 5折交叉验证 + GridSearch |
| LightGBM | num_leaves, learning_rate, n_estimators | 5折交叉验证 + GridSearch |
| 岭回归(Ridge) | alpha | 5折交叉验证 |

**数据划分策略：**
- 按时间划分：2011-2020训练集，2021-2023测试集
- 避免数据泄露：所有缩放/编码均在训练集上fit，再transform测试集

#### B3. 评价指标（模型对比表）

| 评价指标 | 公式 | 说明 |
|---------|------|------|
| R² | $1 - \frac{SS_{res}}{SS_{tot}}$ | 解释方差比例（越大越好） |
| RMSE | $\sqrt{\frac{1}{n}\sum(y_i - \hat{y}_i)^2}$ | 均方根误差（越小越好） |
| MAE | $\frac{1}{n}\sum\|y_i - \hat{y}_i\|$ | 平均绝对误差（越小越好） |
| MAPE | $\frac{1}{n}\sum\|\frac{y_i - \hat{y}_i}{y_i}\| \times 100\%$ | 平均百分比误差 |

#### B4. SHAP可解释性分析（论文核心亮点）

使用最优ML模型（预计为XGBoost或LightGBM），计算SHAP值：

**必须呈现的4张SHAP图：**

| 图表 | 内容 | 论文中的作用 |
|------|------|-------------|
| **① SHAP蜂群图** (Beeswarm) | 所有特征的SHAP值分布 | 全局特征重要性排序（替代传统"系数解读"） |
| **② SHAP柱状图** (Bar) | 平均绝对SHAP值排序 | 直观展示"哪些数字经济因素最影响碳排放" |
| **③ SHAP依赖图** (Dependence) | 单个特征值 vs SHAP值的散点 | 揭示非线性关系（与门槛效应交叉验证） |
| **④ SHAP交互图** (Interaction) | 两特征的交互SHAP值 | 发现统计回归未捕捉到的交互效应 |

**关键论述角度：**
```
"统计回归模型估计了数字经济对碳排放的平均边际效应（回归系数），
 而SHAP分析揭示了每个样本层面的个体化影响方向与强度。
 两者互补：回归回答'是否影响'，SHAP回答'如何影响'和'影响多大'。
 例如，SHAP依赖图显示互联网普及率在超过XX%后，其减碳边际效应显著增强，
 这与Hansen门槛模型估计的门槛值XX一致，形成了交叉验证。"
```

### 3.4 稳健性检验方案（7项）

| 序号 | 检验方法 | 具体操作 |
|------|---------|---------|
| 1 | 替换因变量 | 用人均碳排放替代碳排放强度 |
| 2 | 替换自变量构建法 | PCA构建数字经济指数替代熵权法 |
| 3 | 工具变量法(IV) | 各省1984年固定电话数量作为工具变量 |
| 4 | 系统GMM | 动态面板估计，处理内生性 |
| 5 | 替换空间权重矩阵 | 经济距离矩阵替代地理邻接矩阵 |
| 6 | 缩尾处理 | 所有连续变量1%和99%缩尾 |
| 7 | 排除直辖市 | 剔除北京、天津、上海、重庆 |

## 四、数据来源与变量设计

### 4.1 数据范围

- **空间范围**：中国30个省级行政区（除港澳台及西藏）
- **时间范围**：2011-2023年（13年，390个观测值）

### 4.2 变量一览表

| 类型 | 变量名 | 定义 | 数据来源 |
|------|--------|------|---------|
| **因变量** | 碳排放强度(CI) | CO2排放量/GDP | CEADs (ceads.net.cn) |
| **因变量（稳健性）** | 人均碳排放 | CO2排放量/常住人口 | CEADs + 国家统计局 |
| **核心自变量** | 数字经济指数(DE) | 熵权法构建（4维12指标） | 多源整合 |
| 控制变量 | 人均GDP (PGDP) | ln(人均GDP) | 国家统计局 |
| 控制变量 | 产业结构 (IS) | 第二产业增加值/GDP | 国家统计局 |
| 控制变量 | 城镇化率 (URB) | 城镇常住人口比例 | 各省统计年鉴 |
| 控制变量 | 对外开放 (FDI) | 实际利用外资/GDP | 商务部 |
| 控制变量 | 环境规制 (ER) | 环境污染治理投资/GDP | 环境统计年鉴 |
| 控制变量 | 能源结构 (ES) | 煤炭消费/一次能源总量 | 中国能源统计年鉴 |
| 中介变量 | 产业结构高级化(ISU) | 第三产业/第二产业 | 国家统计局 |
| 中介变量 | 绿色技术创新(GTI) | ln(绿色专利申请数) | CNIPA |
| 中介变量 | 能源效率(EE) | GDP/能源消费总量 | 中国能源统计年鉴 |
| 门槛变量 | 人均GDP | 同上 | 国家统计局 |
| 工具变量 | 1984年固话数 | 各省1984年固定电话用户 | 历史统计年鉴 |

### 4.3 数字经济综合指数构建体系

| 一级指标（权重待定） | 二级指标 | 数据来源 |
|-------------------|---------|---------|
| 数字基础设施 | 互联网普及率、移动电话普及率、光缆线路长度 | 中国统计年鉴 |
| 数字产业发展 | 电信业务总量、信息技术服务收入、软件业务收入 | 工信部统计公报 |
| 数字金融发展 | 覆盖广度、使用深度、数字化程度 | 北大数字金融中心(PKU-DFIIC) |
| 数字创新能力 | 数字技术专利数、R&D人员密度、技术市场成交额 | CNIPA、科技统计年鉴 |

> 注意：在ML模型中，直接使用12个二级指标作为特征输入（而非综合指数），以获得更细粒度的SHAP分析结果。

### 7.2 Python ML核心代码框架（参考）

```python
# ===== 1. 数据准备 =====
import pandas as pd
import numpy as np
from sklearn.model_selection import GridSearchCV, TimeSeriesSplit
from sklearn.ensemble import RandomForestRegressor
from sklearn.linear_model import Ridge
from xgboost import XGBRegressor
from lightgbm import LGBMRegressor
from sklearn.metrics import r2_score, mean_squared_error, mean_absolute_error
import shap
import matplotlib.pyplot as plt

# 读取面板数据
df = pd.read_csv('merged_panel_data.csv')

# 特征：12个数字经济子指标 + 6个控制变量
features = [
    '互联网普及率', '移动电话普及率', '光缆长度',
    '电信业务总量', 'IT服务收入', '软件业务收入',
    '金融覆盖广度', '金融使用深度', '金融数字化程度',
    '数字专利数', 'RD人员密度', '技术市场成交额',
    'ln人均GDP', '产业结构', '城镇化率', 'FDI', '环境规制', '能源结构'
]
target = '碳排放强度'

# 按时间划分训练/测试集
train = df[df['年份'] <= 2020]
test  = df[df['年份'] >= 2021]

X_train, y_train = train[features], train[target]
X_test,  y_test  = test[features],  test[target]

# ===== 2. 模型训练（以XGBoost为例）=====
xgb_params = {
    'max_depth': [3, 5, 7],
    'learning_rate': [0.01, 0.05, 0.1],
    'n_estimators': [100, 200, 300],
    'reg_alpha': [0, 0.1, 1]
}
xgb = XGBRegressor(random_state=42)
grid = GridSearchCV(xgb, xgb_params, cv=5, scoring='r2', n_jobs=-1)
grid.fit(X_train, y_train)
best_xgb = grid.best_estimator_

# ===== 3. 模型对比（论文表5）=====
models = {
    'Ridge': Ridge(),
    'RandomForest': RandomForestRegressor(n_estimators=200, random_state=42),
    'XGBoost': best_xgb,
    'LightGBM': LGBMRegressor(n_estimators=200, random_state=42)
}
results = []
for name, model in models.items():
    if name != 'XGBoost':
        model.fit(X_train, y_train)
    y_pred = model.predict(X_test)
    results.append({
        '模型': name,
        'R²': round(r2_score(y_test, y_pred), 4),
        'RMSE': round(np.sqrt(mean_squared_error(y_test, y_pred)), 4),
        'MAE': round(mean_absolute_error(y_test, y_pred), 4)
    })
print(pd.DataFrame(results))

# ===== 4. SHAP分析（论文图8-11）=====
explainer = shap.TreeExplainer(best_xgb)
shap_values = explainer.shap_values(X_test)

# 图8: SHAP蜂群图
plt.figure(figsize=(10, 8))
shap.summary_plot(shap_values, X_test, feature_names=features, show=False)
plt.tight_layout()
plt.savefig('fig8_shap_beeswarm.png', dpi=300)

# 图9: SHAP柱状图
plt.figure(figsize=(10, 6))
shap.summary_plot(shap_values, X_test, feature_names=features, 
                  plot_type='bar', show=False)
plt.tight_layout()
plt.savefig('fig9_shap_bar.png', dpi=300)

# 图10: SHAP依赖图（以互联网普及率为例）
shap.dependence_plot('互联网普及率', shap_values, X_test, 
                     feature_names=features)
plt.savefig('fig10_shap_dependence.png', dpi=300)
```
